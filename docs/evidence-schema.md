# Evidence Schema Notes

These notes describe the evidence JSON shape produced by the current synthetic demo export.
They are intended for review and discussion, not as a stable package API.

An individual scenario export is written by:

```bash
python3 -m notary_memory_kit.cli export demo/atlas-docs-migration --notary out/notary-evidence.json
```

## Top-Level Shape

The exported JSON contains three top-level keys:

```json
{
  "facts": [],
  "authorities": [],
  "authority_audit": []
}
```

- `facts` contains normalized memory evidence records loaded from the synthetic fixture.
- `authorities` contains the fixture's write-authority records.
- `authority_audit` contains warnings found by the kit before Notary scoring.

## Fact Records

Each fact record must include:

| Field | Type | Notes |
| --- | --- | --- |
| `fact_id` | string | Unique fact identifier within the export. |
| `content` | string | The evidence text being reviewed. |
| `agent_id` | string | Synthetic agent that wrote the fact. |
| `session_id` | string | Synthetic session identifier. |
| `timestamp` | ISO-like string | Must parse as an ISO timestamp and include a `T` separator. |
| `surface` | string | The write surface used by the fact. |
| `lifecycle` | string | One of `permanent`, `session`, or `volatile`. |
| `confidence` | number or numeric string | Must be between `0` and `1`, inclusive. |

The current demo also uses optional fields:

| Field | Type | Notes |
| --- | --- | --- |
| `overwrite_of` | string or null | Points to the fact this record attempts to update or overwrite. |
| `tags` | array of strings | Simple labels used by the demo search flow. |
| `evidence_path` | string | Relative path to the synthetic source file. Absolute paths and parent traversal are rejected by the demo test fixture. |

## Authority Records

Authority records are loaded from `policy/write-authority.json`.
Each authority record must include:

| Field | Type | Notes |
| --- | --- | --- |
| `agent_id` | string | Synthetic agent covered by the authority rule. |
| `allowed_surfaces` | array of strings | Surfaces the agent is allowed to write to. |
| `can_overwrite` | boolean | Whether the agent is allowed to overwrite existing facts. |

Every fact with an `agent_id` must have a matching authority record.

## Authority Audit Warnings

`authority_audit` is a list of pre-scoring warnings generated by the kit.
The current audit checks whether a fact was written to a surface that is not listed in the writer's `allowed_surfaces`.

The synthetic scenarios intentionally include authority warnings. For example, the Atlas scenario includes:

```text
[atlas-f005] agent 'builder' is not authorized for surface 'project_scope'
```

These warnings are deliberate. They show that the kit can preserve the evidence while flagging governance concerns for review.

## Validation Behavior

The current demo validation fails ingestion when:

- a required fact field is missing;
- a `fact_id` is duplicated;
- `lifecycle` is not `permanent`, `session`, or `volatile`;
- `timestamp` is not ISO-like;
- `confidence` is non-numeric or outside `0..1`;
- an authority record has invalid `allowed_surfaces`;
- an authority record has non-boolean `can_overwrite`;
- a fact references an `agent_id` with no authority record.

Authority-surface mismatches are warnings, not ingestion failures, because the demo is designed to keep reviewable evidence intact.

## Boundary

This schema describes a synthetic evidence-preparation demo.
It does not define a production memory model, a retrieval API, a hosted service contract, or a package compatibility guarantee.
